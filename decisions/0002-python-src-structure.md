## 컨텍스트

Python 프로젝트들의 구조가 일관되지 않았습니다:
- 일부는 루트에 `main.py` 직접 배치
- 일부는 `app/` 디렉토리 사용
- 일부는 `src/` 디렉토리 사용

**문제점:**
- 프로젝트마다 import 방식이 다름
- 패키지 설치 시 경로 문제 발생
- 테스트와 소스 코드 분리 어려움

## 결정

**모든 Python 프로젝트는 `src/` 패키지 구조를 사용**합니다.

```
project/
├── src/
│   ├── __init__.py
│   ├── main.py      # 진입점
│   ├── config.py    # 설정
│   └── api/         # 추가 모듈
├── tests/
├── requirements.txt
└── pyproject.toml
```

실행 방법:
```bash
uvicorn src.main:app --reload
```

## 고려한 대안

### 대안 1: 루트에 main.py 배치

- **장점**: 단순함, 빠른 시작
- **단점**: 패키지로 import 어려움, 테스트 설정 복잡
- **불채택 사유**: 프로젝트 규모 커지면 문제 발생

### 대안 2: app/ 디렉토리 사용

- **장점**: Flask 커뮤니티에서 흔히 사용
- **단점**: src/ layout의 이점을 못 누림
- **불채택 사유**: Python 패키징 표준인 src layout 선호
